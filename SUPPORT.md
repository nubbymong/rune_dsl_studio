# Support

Thanks for using **Rune DSL Studio**.

## Getting help
- **In-app feedback (fastest):** Command Palette → **"Rune DSL: Send Feedback…"**.
  It attaches the diagnostics you can review first; reports go to a private maintainer
  repository and are never published publicly.
- **Issues / bugs:** https://github.com/nubbymong/rune_dsl_studio/issues

## Before you file
- Check the environment checklist in the Studio welcome view (Java 21, Maven 3.6.3+,
  Python 3.10+, GitHub sign-in) — most build problems are a missing prerequisite with
  a one-click fix.
- Include your Rune DSL Studio / VS Code / OS versions (shown in the Studio footer).

## The underlying language
Rune DSL Studio is a workbench around the open-source **[Rune DSL](https://github.com/finos/rune-dsl)**
language — a [FINOS](https://www.finos.org/) (Fintech Open Source Foundation) project,
maintained by Regnosys. Questions about the DSL itself, the CDM, or DRR modelling are
best directed to those upstream communities.
